## 1. 项目概述

### 1.1 产品定位
用户管理系统是一套基于 Web 的全栈解决方案，提供用户认证和完整的用户管理功能。该系统采用前后端分离架构，适用于 API 自动化测试演示、学习 RESTful API 开发，以及作为前端开发的模拟后端服务。

### 1.2 目标用户
- 开发人员进行 API 自动化测试
- 学习前后端分离架构的开发者
- 需要快速搭建用户管理原型的团队

### 1.3 核心价值
- **快速部署**：开箱即用，无需复杂配置
- **功能完整**：包含完整的用户 CRUD 操作
- **技术先进**：采用 Vue 3 + Flask 现代化技术栈
- **易于扩展**：清晰的代码结构，便于二次开发

## 2. 功能需求

### 2.1 用户认证模块

#### 2.1.1 用户登录
**功能描述**：用户通过用户名和密码进行身份认证

**输入**：
- 用户名（必填）
- 密码（必填）

**处理逻辑**：
1. 验证用户名和密码是否为空
2. 查询用户数据库匹配用户名和密码
3. 验证成功后生成唯一 Token
4. 将 Token 与用户 ID 关联存储

**输出**：
- 成功：返回 Token 和用户 ID
- 失败：返回错误信息（用户名或密码错误）

**接口**：`POST /auth/login`

**响应示例**：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "token": "uuid-string",
    "userId": "1"
  }
}
```

#### 2.1.2 退出登录
**功能描述**：用户退出系统，清除本地存储的 Token

**处理逻辑**：
1. 清除 localStorage 中的 Token
2. 重置用户状态
3. 跳转到登录页面

### 2.2 用户管理模块

#### 2.2.1 获取当前用户信息
**功能描述**：获取当前登录用户的详细信息

**输入**：
- Authorization Header: Bearer Token

**处理逻辑**：
1. 验证 Token 有效性
2. 根据 Token 获取用户 ID
3. 查询用户详细信息

**输出**：用户详细信息（ID、用户名、邮箱）

**接口**：`GET /api/v1/user`

**响应示例**：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "userId": "1",
    "username": "admin",
    "email": "admin@demo.com"
  }
}
```

#### 2.2.2 获取用户列表
**功能描述**：获取系统中所有用户的基本信息列表

**输入**：
- Authorization Header: Bearer Token

**处理逻辑**：
1. 验证 Token 有效性
2. 查询所有用户
3. 返回用户基本信息（不包含密码）

**输出**：用户列表和总数

**接口**：`GET /api/v1/users`

**响应示例**：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "list": [
      {
        "id": "1",
        "username": "admin",
        "email": "admin@demo.com"
      }
    ],
    "total": 2
  }
}
```

#### 2.2.3 创建用户
**功能描述**：创建新的用户账户

**输入**：
- Authorization Header: Bearer Token
- 用户名（必填）
- 密码（必填）
- 邮箱（可选）

**处理逻辑**：
1. 验证 Token 有效性
2. 验证用户名和密码是否为空
3. 检查用户名是否已存在
4. 生成唯一用户 ID
5. 保存用户信息

**输出**：创建成功的用户信息

**接口**：`POST /api/v1/users`

**请求示例**：
```json
{
  "username": "newuser",
  "password": "pass123",
  "email": "new@demo.com"
}
```

#### 2.2.4 获取指定用户
**功能描述**：获取指定用户的详细信息

**输入**：
- Authorization Header: Bearer Token
- 用户 ID（路径参数）

**处理逻辑**：
1. 验证 Token 有效性
2. 根据用户 ID 查询用户信息
3. 返回用户详细信息

**输出**：用户详细信息

**接口**：`GET /api/v1/users/{user_id}`

#### 2.2.5 更新用户信息
**功能描述**：更新指定用户的信息

**输入**：
- Authorization Header: Bearer Token
- 用户 ID（路径参数）
- 邮箱（可选）
- 密码（可选）

**处理逻辑**：
1. 验证 Token 有效性
2. 验证用户是否存在
3. 更新邮箱和密码字段
4. 返回更新后的用户信息

**输出**：更新后的用户信息

**接口**：`PUT /api/v1/users/{user_id}`

#### 2.2.6 删除用户
**功能描述**：删除指定用户及其相关 Token

**输入**：
- Authorization Header: Bearer Token
- 用户 ID（路径参数）

**处理逻辑**：
1. 验证 Token 有效性
2. 验证用户是否存在
3. 删除用户信息
4. 清理该用户的所有 Token
5. 返回删除结果

**输出**：删除成功确认

**接口**：`DELETE /api/v1/users/{user_id}`

## 5. 接口规范

### 5.1 统一响应格式
所有接口返回统一的 JSON 格式：
```json
{
  "code": number,      // 状态码
  "msg": string,       // 消息描述
  "data": any          // 响应数据
}
```

### 5.2 状态码说明
| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权/认证失败 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 5.3 认证规范
- 需要认证的接口必须在请求头中包含 `Authorization`
- 格式：`Authorization: Bearer <token>`
- Token 由登录接口返回

## 6. 部署说明

### 6.1 环境要求
- Python 3.8+
- Node.js 16+
- 现代浏览器（Chrome、Firefox、Safari、Edge）

### 6.2 后端部署

#### 安装依赖
```bash
pip install -r requirements.txt
```

#### 启动服务
```bash
python app.py
```

#### 配置说明
- **默认端口**：11011
- **环境变量**：`DEMO_PORT` 可自定义端口
- **调试模式**：开启（生产环境应关闭）

### 6.3 前端部署

#### 安装依赖
```bash
npm install
```

#### 开发模式
```bash
npm run dev
```
- 开发服务器端口：3000
- 自动代理到后端服务

#### 生产构建
```bash
npm run build
```
- 构建输出目录：`dist/`

## 7. 安全考虑

### 7.1 当前实现
- Token 认证机制
- 密码验证（明文存储，仅用于演示）
- 请求头认证

### 7.2 生产环境建议
- 使用 HTTPS 加密传输
- 密码加密存储（bcrypt、argon2 等）
- Token 过期机制
- 防止暴力破解（登录次数限制）
- CORS 配置
- 输入验证和 XSS 防护
- SQL 注入防护（如使用数据库）
